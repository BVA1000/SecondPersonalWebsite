State Tax Information

Something that is more simple is this application. I was inspired by this idea when I was moving to Montata several months ago. It is called State Tax Information, and is a simple HTML and CSS application with a main page which includes links to all fifty states in the United States with tax rate information for each state. The idea behind this was to create a simple application where the user can see which state is the best option for them in terms of tax savings.

As you can see in the next photo when you click on one of the anchor tag links, like Texas, you will see four tax categories: State Individual Income Tax Rate, Corporate Income Tax Rate, Sales Tax Rate, and Property Tax Rate. When it comes to doing research on state tax rates, these are the four main categories that are most important when determining the states with the lowest and highest tax rates.

I created this application without the knowledge of frontend frameworks such as React and Angular, so there is not much to it. If I have spare time I will add images and some JavaScript functionality so that it is not just HTML and CSS, but it was just a quck and easy project I did in my spare time.

Ionic Book App

For me I would say this is one of the most fun projects that I have worked on. One of the reasons being is I learned how to integrate Ionic and Cordova into an Angular project. What is so great about Ionic and Cordova is that they are UI frameworks which are used for building high quality mobile and desktop apps. So with this project I was able to make it not only the desktop user has a layout, but also have one for the mobile and tablet users. The idea behind the project was to create a photo book application which would include pictures of my most favorite photos that I have taken with my iPhone camera and would also include a short story about them. Let's go have a look at it.

Here are four images of what the application would look like on a mobile device such as an iPhone. On the far left is the image of what the "Home" page looks like for the album. What I wanted to do with the background of every card is to have it match with the photo so that every page has a different color based on it's colors matching with the background. On the mid left is the image of the "Table Of Contents". This page has a list of seven pages for the album, which are links that are utilized by a TypeScript function that stores routes to every page so that when the user clicks on a section or page in the "Table Of Contents", it directs them to that exact page that the user wants to view.

The next two images display one of the pages to the album. For the first image to the mid right it has the page number on the top left, with the title and subtitle, the photograph, and the description of the work. Once the user scrolls down the page, on the second image to the far right, is displayed the "Table Of Contents" and the "Back" and "Next" buttons. What I thoght would be good besides just creating basic navigation for pages was to also include for the user an option to go back to the "Table Of Contents" page. This way the user has an option to redirect back to the "Table Of Contents" page where the user can choose the desired page that they would like to be directed to.

The photo above is the desktop version of the "Home" page. This was a fun and interestion project for me. Even though it is just a start to my knowledge of creating a good format for multiple devices, I am happy with my results.
